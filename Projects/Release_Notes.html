Release Notes for

Projects of STM32CubeN6 Firmware Package

Copyright © 2024 STMicroelectronics

Purpose

The projects are a set of applicative tasks that aim to explain and provide use cases of the different products features based on the products hardware (Boards, STM32 Core(s) features, interconnections and peripherals) and built around the different firmware components. They can be directly used or adapted by the STM32CubeN6 Firmware Package users. They provide basic functionalities, show main features or demonstrate performance. They are provided with preconfigured projects for the main supported toolchains.

In the STM32CubeN6 Firmware Package, the projects are organized in a sequential scheme based on the following levels:

  • Boards: Hardware platform (evaluation, discovery, nucleo boards)
  • Project Category:
    • Applications
    • Examples
    • Examples_LL
    • Examples_MIX
    • ROT_Provisioning
    • Templates
    • Templates_LL
    • Templates_Board
  • Project Class
  • Project Subclass

The Project Class refers to the main peripherals (drivers), middleware libraries or BSP classes while the projects Subclass stands for a project specific feature within a class.

The exhaustive list of projects is provided in this table: STM32CubeProjectsList.

Notes

BootROM in development mode / BootROM in boot from external Flash mode

  • To set the bootROM in development mode for debug or for binary download in internal SRAM or external Flash,
    • BOOT1 switch position should be 1-3 on STM32N6570-DK board, BOOT0 switch position doesn’t matter,
    • BOOT1 switch position should be 2-3 on NUCLEO-N657X0-Q board, BOOT0 switch position doesn’t matter.
  • To set the bootROM in boot from external Flash mode,
    • BOOT0 and BOOT1 switches positions should be 1-2 on STM32N6570-DK board,
    • BOOT0 and BOOT1 switches positions should be 1-2 on NUCLEO-N657X0-Q board.
  • OTP bits HSLV_VDDIO2 and HSLV_VDDIO3 must imperatively be set if STM32N6570-DK board external memories are used.
  • OTP bit HSLV_VDDIO3 must imperatively be set if NUCLEO-N657X0-Q board external memory is used.
    • User may resort to example Projects\STM32N6570-DK\Examples\BSEC\BSEC_OTP_Config to set OTP bits.

Clocks speed

  • Projects are provided with a configuration defined for nominal voltage mode and are setting the CPU clock at 600 MHz and the System clock at 400 MHz.
  • An example of overdrive voltage configuration, setting the CPU clock at 800 MHz and the System clock at 400 MHz can be found in the project Projects\STM32N6570-DK\Templates\Template.


Known Limitations

  • The NetXDuo Nx_Iperf application running on the STM32N6570-DK board, which embeds a Gigabit Ethernet PHY, does not achieve the expected maximum network throughput. The highest data rates achieved reach around 250 Mb.

  • The NetXDuo Nx_Iperf application occasionally gives low bandwidth during TCP transmit test. This instability will be addressed in future releases.

  • A Hardfault_handler() occurs with USBX applications in CubeIDE Debug mode. This issue is resolved by increasing the optimization level to -O3 for the file stm32n6xx_hal_rcc_ex.c.

  • Some projects are not available on MDK-ARM or CubeIDE.

  • Some projects are not generated with STM32CubeMX tool. Please refer to (STM32CubeProjectsList.html).
